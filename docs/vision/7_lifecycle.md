# ACCU — Project Lifecycle

## Purpose of This Document

This document defines the lifecycle of a project within the ACCU ecosystem — from discovery to long-term sustainability.

---

## Lifecycle Phases

### Phase 0 — Discovery

**Inputs:**
- AI Scout signals
- Community nominations

**Outputs:**
- Candidate listing
- Initial metadata snapshot

No commitments are made at this stage.

---

### Phase 1 — Evaluation

**Activities:**
- Technical assessment
- License verification
- Product re-evaluation
- Risk analysis

**Decision Gate:**
- Curator nomination
- Steward approval

---

### Phase 2 — Onboarding

**Activities:**
- Author outreach
- Governance setup
- Initial share allocation

Projects formally enter ACCU here.

---

### Phase 3 — Modernization

**Activities:**
- Refactoring
- Stack updates
- Documentation
- Infrastructure setup

Contributions during this phase generate CU and PS.

---

### Phase 4 — Production & Growth

**Activities:**
- Feature evolution
- User onboarding
- Revenue activation

Governance becomes project-centric.

---

### Phase 5 — Sustainability

**Characteristics:**
- Stable maintenance cadence
- Reduced intervention
- Ongoing revenue distribution

Projects may remain indefinitely in this phase.

---

### Phase 6 — Sunset or Fork

**Conditions:**
- Declining relevance
- Governance deadlock
- Strategic divergence

Outcomes:
- Graceful archival
- Community fork
- License-respecting exit

---

## Exit Guarantees

- No forced shutdowns
- Author rights preserved
- Transparent rationale

---

## Summary

ACCU treats software as a living system.

Projects are allowed to mature, stabilize, or conclude without coercion.
